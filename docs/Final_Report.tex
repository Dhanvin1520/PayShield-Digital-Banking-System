% --- 1. Executive Summary ---
\section{Executive Summary}
\begin{tcolorbox}[colback=primary!5,colframe=primary,title=Abstract]
PayShield is a production-grade digital banking ecosystem designed to showcase the integration of advanced software engineering principles with modern web technologies. Built using the MERN stack with TypeScript, the platform features a real-time Fraud Detection Engine powered by GoF Design Patterns. This report details the architectural decisions, design patterns, and system requirements that define PayShield as a secure and scalable financial solution.
\end{tcolorbox}

% --- 2. Project Overview ---
\section{Project Overview}
The primary objective of PayShield is to bridge the gap between simple CRUD applications and enterprise-grade financial systems. By implementing a rule-based fraud detection pipeline, the system ensures that every fund transfer is scrutinized before execution, significantly reducing the risk of malicious activity.

% --- 3. Requirements & Use Cases ---
\section{System Requirements \& Use Case Analysis}
The system serves two distinct user roles with specific functionalities.

\subsection{Customer Use Cases}
\begin{itemize}[label=\checkmark]
    \item \textbf{Identity Management:} Secure registration and authentication using JWT-based session management.
    \item \textbf{Account Orchestration:} Opening specialized Savings and Checking accounts with automated logic.
    \item \textbf{Fund Movement:} Real-time peer-to-peer transfers with balance integrity checks.
    \item \textbf{Credit Services:} Application for loans with automated EMI calculation and status tracking.
\end{itemize}

\subsection{Administrator Use Cases}
\begin{itemize}[label=\checkmark]
    \item \textbf{Fraud Oversight:} Monitoring and investigating transactions flagged by the Strategy-based engine.
    \item \textbf{Loan Lifecycle Management:} Reviewing, approving, and rejecting customer loan applications.
    \item \textbf{System Audit:} Access to global account data and system liquidity metrics.
\end{itemize}

% --- 4. Technical Architecture ---
\section{Technical Architecture}
The system follows a strict \textbf{3-Tier Layered Architecture}:
\begin{enumerate}
    \item \textbf{Presentation Layer:} React.js + Tailwind CSS providing a premium, responsive UI.
    \item \textbf{Business Logic Layer:} Node.js + Express + TypeScript implementing core banking services and design patterns.
    \item \textbf{Data Access Layer:} MongoDB Atlas providing a flexible, document-based cloud storage solution.
\end{enumerate}

% --- 5. Design Pattern Implementation ---
\section{Design Pattern Implementation}
The core strength of PayShield lies in its utilization of five Gang of Four (GoF) Design Patterns.

\subsection{Creational Patterns}
\begin{description}
    \item[Singleton Pattern:] Implemented in \texttt{DatabaseConnection.ts} to ensure a single, efficient MongoDB connection pool across the application lifecycle.
    \item[Factory Pattern:] Utilized in \texttt{AccountFactory.ts} to abstract the creation of diverse account types (Savings/Checking) with their respective interest and overdraft rules.
\end{description}

\subsection{Behavioral Patterns}
\begin{description}
    \item[Strategy Pattern:] The foundation of the Fraud Engine. Different detection rules (HighValue, RapidTxn, etc.) are implemented as interchangeable strategies.
    \item[Observer Pattern:] Decouples the Fraud Engine from the Alert system. When fraud is detected, an event is emitted, and the alert observer handles the DB persistence.
    \item[Command Pattern:] Encapsulates fund transfers as objects, enabling transaction history logging and potential multi-step rollbacks.
\end{description}

% --- 6. Software Engineering Principles ---
\section{Software Engineering Principles}
\subsection{SOLID Compliance}
\begin{itemize}
    \item \textbf{Single Responsibility:} Each service is dedicated to one domain (Auth, Account, Loan).
    \item \textbf{Open/Closed:} The Fraud Engine is open for new strategies but closed for modification of core logic.
    \item \textbf{Dependency Inversion:} Controllers depend on high-level Service abstractions rather than low-level Mongoose models.
\end{itemize}

\subsection{OOP Integration}
The codebase utilizes \textbf{Encapsulation} for data protection, \textbf{Inheritance} for account types, and \textbf{Polymorphism} for uniform rule execution across the fraud pipeline.

% --- 7. Conclusion ---
\section{Conclusion}
PayShield successfully demonstrates that modern web frameworks, when paired with classical design patterns, can produce a robust and secure banking environment. The project meets all functional requirements while maintaining a high level of architectural integrity, making it a viable template for larger-scale financial systems.
